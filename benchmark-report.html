02 — Speculative Decoding

Why Different Models Fail as Drafts

Speculative decoding uses a small draft model to predict tokens in advance, verified in parallel by the target. It only helps if the draft's predictions match the target's distribution.

Result — 0% Acceptance Rate

E4B target + E2B draft produced zero accepted draft tokens across 3 runs (184 total draft tokens). The models share a family name but have fundamentally different output distributions. Every draft token was rejected, making spec decoding actively harmful.

Why It Failed

The E2B (5B params) and E4B (8B params) models are architecturally similar but behaviorally different. Despite sharing the Gemma 4 family, the smaller model's predictions diverge significantly — it has different learned patterns, different attention heads, and different top-1 token preferences. Spec decoding requires near-identical output distributions, not just similar architectures.

03 — TurboQuant Self-Drafting

Same Model, Lower Precision

The TurboQuant insight: instead of using a different model as draft, use the same model at much lower quantization. Same weights, same distribution, much higher acceptance rate. Theoretically elegant — but VRAM is the enemy.

Effective Speed Calculation
T_cycle = K / S_draft + K / S_target
Output = M + 1   // accepted tokens + 1 verified
S_effective = (M + 1) / T_cycle

// With CPU draft at 15 tok/s, GPU target at 67 tok/s, K=8, M=5:
T_cycle = 8/15 + 8/67 = 0.533 + 0.119 = 0.653s
S_effective = 6 / 0.653 = 9.2 tok/s   // 7× slower than solo!
Mathematical Proof of Failure

Even with 80% acceptance rate, the CPU draft at ~15 tok/s becomes the bottleneck. The GPU target (67 tok/s) spends most of its time waiting for the draft. Net result: 9.2 tok/s — 7× slower than running the target alone. For spec decoding to help, the draft must be faster than the target.

04 — Quantization Findings

Choosing the Right Precision

On 6GB VRAM, quantization isn't a quality trade-off — it's a survival requirement. The right quantization format determines whether a model fits on GPU at all.

Optimal Sweet Spot — Q4_K_M

Q4_K_M hits the balance: good quality, fits in 6GB with room for KV cache. For Gemma4 E4B (8B params), Q4_K_M = 4.98 GB, leaving ~1.2 GB for KV cache and overhead. VRAM usage: 3821 MiB — 62% of available.

KV Cache Quantization

Using --cache-type-k q4_0 --cache-type-v q4_0 compresses the KV cache to 4-bit precision. This reduces memory per token by ~4×, allowing longer context windows within the same VRAM budget. Impact on quality is minimal for short contexts (<4096 tokens).

06 — Conclusions

What We Learned

Seven models, three spec decoding configurations, and a dozen quantization tests. Here's what matters for consumer GPU inference.

01

Solo Wins on Constrained VRAM

With 6GB VRAM, running a single model with all layers on GPU at 67 tok/s beats any speculative decoding setup. The GPU is fast enough on its own — adding a CPU draft only introduces latency.

02

Spec Decoding Needs GPU Draft

Speculative decoding only helps when the draft model is faster than the target. On consumer hardware, this means both models must fit on GPU simultaneously — which requires ≥12GB VRAM for 8B-class models.

03

Quantization > Inference Tricks

Choosing the right GGUF format (Q4_K_M) and enabling KV cache quantization (q4_0) delivers more real-world improvement than speculative decoding, flash attention, or any other inference optimization on this hardware class.

04

Different Models ≠ Good Drafts

Even models from the same family (Gemma4 E2B/E4B) can have 0% acceptance rates. The TurboQuant insight — same model, lower precision — is theoretically sound but impractical when both can't fit on GPU.

Final Recommendation

For the RTX 3060 6GB: Gemma4 E4B Q4_K_M with all GPU layers, quantized KV cache, and flash attention.

llama-server -m gemma-4-E4B-it-Q4_K_M.gguf -ngl 99 -c 2048 -t 8 --flash-attn auto --cache-type-k q4_0 --cache-type-v q4_0 --host 0.0.0.0 --port 8201

Result: 67 tok/s generation, ~71 tok/s prompt processing. This is the ceiling for 8B-parameter models on 6GB VRAM.
